# Product Requirements Document — MovieBox Bot + Link Server

## Overview

A two-part system:
1. **SKDL Bot** (`@SK_DLBOT`) — A Telegram bot that lets users request movies and TV series using natural language, powered by Groq AI for intent parsing.
2. **Link Server** (`movies.samkiel.dev`) — A lightweight Next.js server that stores temporary CDN redirect links generated by the bot.

---

## Problem

Accessing movies and TV series programmatically is painful. Existing tools are CLI-only, have no conversational interface, and don't integrate with messaging platforms. This project makes it dead simple: talk to a bot, get your content.

---

## Goals (MVP)

- User can request a movie or series episode in plain English
- Bot understands the request via Groq AI (no need for slash commands)
- Bot fetches content via moviebox-api and delivers:
  - The file directly in Telegram chat (if under 2GB)
  - A short redirect link at `movies.samkiel.dev/[id]`
- Links expire after 6 hours; expired page tells user to re-request
- Bot maintains short conversation history per user for clarification flows

---

## Non-Goals (MVP)

- User accounts or authentication
- Admin dashboard
- Analytics or usage tracking
- Rate limiting (added post-MVP)
- Subtitle language selection (default English only)
- Bulk or season-wide downloads
- Public API for third-party access (future)

---

## Users

- Primary: Personal use + close circle (D3V++ community)
- Secondary: Anyone the bot link is shared with
- Scale assumption: Low traffic MVP, under 50 daily active users

## Features

### Bot (`@SK_DLBOT`)

#### Natural Language Interface
- Every message is processed by Groq (llama-3.3-70b-versatile)
- Groq returns a structured JSON intent object
- Bot acts on the intent — no prefix commands required
- Slash commands still supported as fallback:
  - `/start` — welcome + instructions
  - `/movie <title>` — direct movie request
  - `/series <title> <season> <episode>` — direct series request
  - `/status` — health check

#### Intent Types
| Intent | Description |
|---|---|
| `download_movie` | User wants a movie |
| `download_series` | User wants a specific episode |
| `clarify` | Groq needs more info from user |
| `chat` | General conversation |
| `help` | User needs guidance |

#### Conversation Flow
- Per-user in-memory session stores last 10 messages
- Clarification loop: bot asks, user answers, Groq re-parses with context
- Session cleared after successful download or `/start`

#### Delivery
- Bot attempts to send file directly in chat
- If file exceeds 2GB or upload fails, skips file and sends link only
- Always sends `movies.samkiel.dev/[id]` link regardless
- Message format:
  ```
  🎬 Avatar (2009)
  Quality: 1080p

  📥 Download link: movies.samkiel.dev/x7k2pQr1
  ⏳ Link expires in 6 hours

  [file attached if under 2GB]
  ```

---

### Link Server (`movies.samkiel.dev`)

#### Routes
- `GET /[id]` — main redirect route
  - Looks up ID in Supabase
  - If not found: 404 page
  - If expired: expired page with bot link
  - If valid: 302 redirect to CDN URL

#### Expired Page
- Shows movie/series title
- Clear message: "This link has expired"
- Button: "Request again on Telegram" → links to `t.me/SK_DLBOT`

#### 404 Page
- Simple: "Link not found"
- Button: "Go to bot" → `t.me/SK_DLBOT`

## Data Model

### Supabase — `media` table

```sql
create table media (
  id            text primary key,
  title         text not null,
  cdn_url       text not null,
  type          text not null check (type in ('movie', 'series')),
  quality       text default '1080p',
  season        integer null,
  episode       integer null,
  requested_by  bigint null,        -- telegram user_id
  requested_at  timestamptz default now(),
  expires_at    timestamptz not null
);
```
